# ISO/IEC 42001 實作與證據清單（完整版）

> 本文件列出達成 ISO/IEC 42001 所需的制度、流程與可稽核證據。涵蓋 AIMS 範圍、政策、風險/影響評估、資料/模型治理、監控、事件、供應商、內部稽核與管理審查等。請在完成每一項時保存對應紀錄，以便稽核與管理審查。

## 1. AIMS 基礎
- **範圍聲明**：界定納入的系統、流程、資料、地理與組織邊界；保存版本控管的範圍文件。
- **AI 政策**：責任與倫理原則、可接受風險級別、透明度/公平性/安全性要求；定期審查紀錄。
- **角色與職責**：AI 產品負責人、資料負責人、模型/系統負責人、資安、隱私、合規、Human Oversight；RACI 表與任命紀錄。
- **資源與能力**：能力矩陣、訓練計畫與完成證據；工具/基礎設施清單與授權。

## 2. 風險與影響評估（對應 ISO 23894）
- **方法論**：採用的風險評估流程、分級準則、計分方式，文件化並版控。
- **AI 風險評估報告**：涵蓋模型失效、偏見、資安/隱私、誤用/濫用、可解釋性不足、第三方依賴；含風險處置計畫。
- **AI 影響評估 (AIIA)**：對個人/社會/利害關係人的影響分析與緩解措施。
- **SoA（適用性聲明）**：Annex A 控制的適用/不適用清單與理由，連結風險處置。

## 3. 資料治理
- **資料政策**：來源合法性、品質標準、偏見檢測與緩解、標註規範、留存/刪除策略。
- **來源與血緣紀錄**：資料來源、版次、處理流程、轉換與標註紀錄，可追溯至原始檔。
- **品質與偏見報告**：定期抽樣品質檢測、偏見分析結果與修正措施。
- **隱私與同意**：個資去識別/最小化處理證據，合法依據（同意/契約/法定）紀錄。

## 4. 模型與系統生命週期
- **需求/設計/決策紀錄**：模型選擇理由、目標指標、替代方案評估、預期用途與限制。
- **資料準備與特徵工程紀錄**：含數據切分、版本、前處理步驟。
- **訓練/驗證/測試報告**：指標、交叉驗證、壓力/健壯性/對抗測試、公平性測試、再現步驟。
- **發布/變更管理**：版控策略、審批流程、回滾方案、變更單（含風險/影響評估）。
- **部署與配置紀錄**：環境/參數/權限設定，與基線比對。

## 5. 監控與度量
- **運行監控計畫**：性能、公平性、漂移、延遲、可用性、成本；門檻與警示設定。
- **監控日誌與報表**：定期產出的指標報表、異常調查與修復紀錄。
- **資料/模型漂移檢測**：方法、頻率、結果與處置措施。
- **使用行為監控**：誤用/濫用偵測（如越權查詢），並有封鎖/降級流程。

## 6. 事件與應變
- **事件分類與通報流程**：資安、偏見/倫理、品質/可用性事件的分級、通報與 SLA。
- **事件處理紀錄**：調查、根因分析、矯正/預防措施（CAPA）、回覆利害關係人證據。
- **演練與改進**：事件演練紀錄與流程修訂。

## 7. 第三方與供應商管理
- **供應商評估與選擇**：模型/API/資料供應商的審查準則與結果。
- **合約/條款**：安全、隱私、可用性、支援、審計權、子處理者揭露等要求。
- **持續監控**：版本/可用性/安全通告追蹤，必要時的替代或降級方案。

## 8. 安全與隱私控制（與 ISO 27001/27701 對齊）
- **存取控制**：最小權限、審計日誌、密鑰/金鑰管理、API rate limiting。
- **資料保護**：靜態/傳輸加密、備份與復原、留存與刪除程序。
- **隱私影響評估 (PIA/DPIA)**：若處理個資，提供專屬評估與緩解措施。

## 9. Human Oversight 與透明度
- **人類介入機制**：哪些決策需要人工審核/覆核，如何介入、覆寫或停止系統。
- **使用者告知**：AI 互動揭露、用途/限制、資料使用聲明；對外說明文件或 UI 文案。
- **可解釋性**：提供解釋或溯源能力（引用、檢索來源、決策依據）。

## 10. 溝通、訓練與意識
- **內外部溝通計畫**：更新節點、發布頻率、通報管道。
- **訓練紀錄**：倫理、隱私、安全、產品/流程訓練的完成證據。

## 11. 內部稽核與管理審查
- **內部稽核計畫與報告**：覆蓋風險、控制有效性、紀錄完整性；包含不符合事項與 CAPA。
- **管理審查紀錄**：高層審查 AIMS 效能、風險狀態、資源需求與改進決議。
- **持續改進**：CAPA 追蹤表、完成證據與效果驗證。

## 12. 文件與紀錄管控
- **版控與留存**：政策/程序/紀錄的版本、保存期限、存取控制。
- **稽核追溯**：關鍵紀錄（風險、AIIA、SoA、監控、事件、稽核、管理審查）需可追蹤責任人與時間戳。

## 13. 落地路線圖（建議順序）
1) 範圍聲明 + AI 政策 + 角色/職責 + 版控與紀錄策略
2) 風險/影響評估方法論 → 風險登錄表 → AIIA → SoA
3) 資料治理政策與血緣/品質/偏見紀錄；隱私/合法性確認
4) 模型/系統生命週期流程（需求→訓練→驗證→發布→回滾）與變更管控
5) 監控與度量方案上線，建立報表與警示閉環
6) 事件通報與應變演練；Human Oversight 機制落地
7) 供應商評估與合約條款落實；持續監控
8) 內部稽核計畫、管理審查節奏與 CAPA 追蹤

## 14. 稽核必備證據清單（摘要）
- 範圍/政策/角色文件；訓練與任命紀錄
- 風險評估、AIIA、SoA；風險處置與追蹤
- 資料來源/血緣/品質/偏見報告；隱私與合法性證據
- 模型需求/設計/訓練/驗證/發布/變更紀錄；回滾與復原方案
- 監控報表與異常處置；漂移檢測紀錄
- 事件通報與 CAPA；演練紀錄
- 供應商評估、合約、監控紀錄
- 內部稽核報告、管理審查紀要、CAPA 完成證據
- 文件/紀錄版控與留存策略的執行情形
